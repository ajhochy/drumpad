### 2026-05-20 — Blank-portrait fix in reveal/Drops cards ([PR #8](https://github.com/ajhochy/drumrot/pull/8))
- Symptom: revealed cards rendered chrome correctly but the portrait area showed only the dark gradient — no voxel image, no emoji fallback. Playwright smoke missed it because the check asserted HTTP 200 on the WebP, not painted pixels.
- Cause: `loading="lazy"` on the portrait `img` plus `content-visibility: auto` on the `.cell` wrapper interact badly when the cell mounts inside an opacity:0 ancestor (`#drumrotReveal`). Layout-skip + intersection observer conspire so the image element never reaches a paintable state.
- Fix (belt + suspenders, `e655860` / `5fff600`):
  - Drop `loading="lazy"` from the renderer img (31 cards × ~8KB ≈ 250KB total is well under any eager-load budget).
  - Drop `content-visibility: auto` and `contain-intrinsic-size` from `.cell` — premature optimization that broke the reveal pathway.
  - Always emit emoji `<div class="creature">` behind the image (z-index 0 vs img z-index 2). When the image loads it covers the creature; if the image fails or is skipped, the emoji is the visible fallback. No more silently-blank portraits.
- Smoke checklist (`docs/testing/manual-smoke.md`) now requires a pixel check (rect width > 0, complete, naturalWidth > 0); HTTP 200 is no longer sufficient evidence.
- Added Node static-file dev server (`.claude/serve.js` + `.claude/launch.json`) so the Claude Preview sandbox can serve the static site (`python -m http.server` is blocked by the preview sandbox).
- Postmortem: `.agent-stack/postmortems/2026-05-21-portrait-blank-after-pr5.json` recorded for the self-improvement loop.
- Project state updated post-merge for PR #5 + PR #8 (`5fc31d4` / [PR #11](https://github.com/ajhochy/drumrot/pull/11)).
